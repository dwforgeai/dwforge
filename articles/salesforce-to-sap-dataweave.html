1. Integration overview and architecture

Salesforce to SAP S/4HANA is one of the most common enterprise integration patterns. Salesforce manages the customer-facing side — orders, accounts, service agreements. SAP owns the back office — finance, procurement, inventory, GL posting. MuleSoft sits in the middle, and DataWeave handles the translation between two very different data models.

Integration Architecture — Salesforce to SAP via MuleSoft

2. Understanding the Salesforce payload structure

When MuleSoft queries Salesforce for an Order record, the payload arrives as JSON. Custom fields use the __c suffix. Relationship fields use __r notation and contain nested objects.

{
  "Id": "8014W000002XnzQQAS",
  "Name": "ORD-2024-00123",
  "Status__c": "Activated",
  "EffectiveDate": "2024-06-01",
  "TotalAmount": 15000.00,
  "Account__r": {
    "SAP_Customer_ID__c": "CUST-10045",
    "BillingCountry": "AU"
  },
  "Product__r": {
    "SAP_Material_Number__c": "MAT-88901",
    "ProductCode": "EIS-001"
  },
  "Legal_Entity__c": "AU01",
  "Currency__c": "AUD",
  "Quantity__c": 12.0,
  "TotalAmount": 15000.00,
  "Discount__c": null
}
Watch out
Three fields are null in this payload. Every null field needs explicit handling in DataWeave. SAP will reject a request with an unexpected null where it expects a value.

3. What SAP S/4HANA expects

SAP S/4HANA OData API expects a specific JSON structure for Sales Order creation. Field names are PascalCase, dates must be in ISO 8601 format with time component, and certain fields are mandatory.

{
  "SalesOrderType": "ZOR",
  "SalesOrganization": "AU01",
  "SoldToParty": "CUST-10045",
  "PurchaseOrderByCustomer": "ORD-2024-00123",
  "RequestedDeliveryDate": "2024-06-01T00:00:00",
  "to_Item": {
    "results": [
      {
        "SalesOrderItem": "10",
        "Material": "MAT-88901",
        "RequestedQuantity": "12",
        "RequestedQuantityUnit": "EA",
        "NetAmount": "15000.00",
        "TransactionCurrency": "AUD"
      }
    ]
  }
}

5. Basic DataWeave transformation

Starting simple — a direct field mapping with no null handling. This is the starting point before adding production safeguards.

%dw 2.0
output application/json
---
{
  SalesOrderType: "ZOR",
  SalesOrganization: payload.Legal_Entity__c,
  SoldToParty: payload.Account__r.SAP_Customer_ID__c,
  PurchaseOrderByCustomer: payload.Name,
  RequestedDeliveryDate: payload.EffectiveDate,
  to_Item: {
    results: [{
      SalesOrderItem: "10",
      Material: payload.Product__r.SAP_Material_Number__c,
      RequestedQuantity: payload.Quantity__c,
      RequestedQuantityUnit: "EA",
      NetAmount: payload.TotalAmount,
      TransactionCurrency: payload.Currency__c
    }]
  }
}
Do not use this in production
This will fail the moment any field is null. The next sections add the null handling that makes this production safe.

6. Null handling — the production patterns

SAP is unforgiving with nulls. DataWeave gives you three null handling patterns depending on the business rule.

Pattern 1 — Default value

TransactionCurrency: payload.Currency__c default "AUD"
NetAmount: (payload.TotalAmount default 0) as String

Pattern 2 — Skip the whole record

%dw 2.0
output application/json
---
if (payload.Account__r.SAP_Customer_ID__c != null)
  { SoldToParty: payload.Account__r.SAP_Customer_ID__c }
else {}

Pattern 3 — Conditional field inclusion

(EndDate: payload.EndDate) if (payload.EndDate != null)

7. Date conversion for SAP

SAP OData expects dates in ISO 8601 format with time component. Salesforce Date fields come through as plain date strings.

The most common DataWeave date mistake
Concatenating directly onto a Date type field without casting to String first causes a runtime type error. 
// WRONG — causes runtime type error
RequestedDeliveryDate: payload.EffectiveDate ++ "T00:00:00"

// CORRECT — cast to String first
RequestedDeliveryDate: if (payload.EffectiveDate != null)
  (payload.EffectiveDate as String) ++ "T00:00:00"
else null

8. Salesforce relationship fields

Salesforce relationship fields use the __r notation and return nested objects. The entire relationship object can be null if the lookup field is empty.

// Safe navigation through relationship
SoldToParty: payload.Account__r.SAP_Customer_ID__c default ""

// Explicit null check — clearer intent
SoldToParty: if (payload.Account__r != null and
              payload.Account__r.SAP_Customer_ID__c != null)
  payload.Account__r.SAP_Customer_ID__c
else null

9. Batch transformation

When processing multiple Salesforce records in one flow, the payload arrives as an array. Always filter before map — never chain them directly.

%dw 2.0
output application/json
var validOrders = payload filter (item) ->
  (item.Account__r.SAP_Customer_ID__c != null)
---
validOrders map (order) -> {
  SalesOrderType: "ZOR",
  SalesOrganization: order.Legal_Entity__c default "AU01",
  SoldToParty: order.Account__r.SAP_Customer_ID__c,
  PurchaseOrderByCustomer: order.Name default "",
  RequestedDeliveryDate: if (order.EffectiveDate != null)
    (order.EffectiveDate as String) ++ "T00:00:00"
  else null,
  to_Item: {
    results: [{
      SalesOrderItem: "10",
      Material: order.Product__r.SAP_Material_Number__c default "",
      RequestedQuantity: (order.Quantity__c default 0) as String,
      RequestedQuantityUnit: "EA",
      NetAmount: (order.TotalAmount default 0) as String,
      TransactionCurrency: order.Currency__c default "AUD"
    }]
  }
}
